# Address Module API Documentation

This document describes the request/response structures, authorization, and headers required to interact with the Address Details API.

## Base Configuration

### Required Headers
All requests to these endpoints must include the following headers:

| Header Name | Required | Description | Example |
| :--- | :--- | :--- | :--- |
| `Authorization` | Yes | Bearer/Session identifier token | `Bearer abc123xyz...` |
| `x-app-type` | Yes | Client application context | `web` |
| `accept-language` | Yes | Target response language locale | `en-US` (supports `hi`, `mr`) |

---

## 1. GET Address Details
Retrieve address and location details of a target user.

* **URL**: `/api/auth/profile/address/{userId}`
* **Method**: `GET`
* **Authentication Required**: Yes

### Success Response
* **Status Code**: `200 OK`
* **Format**: Direct flat JSON object (no `statusCode` or `data` wrapper)

```json
{
  "country": "India",
  "state": "Maharashtra",
  "city": "Pune",
  "addressLine1": "ABC Road",
  "addressLine2": "Near Metro Station",
  "pincode": 411033
}
```

---

## 2. POST Create Address Details
Create address details and store user location.

* **URL**: `/api/auth/profile/address`
* **Method**: `POST`
* **Authentication Required**: Yes

### Request Body
* `pincode` is **optional**. If not provided (or empty/whitespace), it defaults internally to `0`.

```json
{
  "country": "India",
  "state": "Maharashtra",
  "city": "Pune",
  "addressLine1": "ABC Road",
  "addressLine2": "Near Metro Station",
  "pincode": 411033
}
```

### Success Response
* **Status Code**: `200 OK`

```json
{
  "statusCode": 200,
  "messageCode": "success.ADDRESS_CREATED",
  "message": "Address details created successfully"
}
```

### Error Responses
* **Status Code**: `400 Bad Request` (Missing fields or invalid JSON)
* **Status Code**: `409 Conflict` (Address already exists for the user)

```json
{
  "statusCode": 409,
  "messageCode": "error.addressDetailsExists",
  "message": "Address details already exist."
}
```

---

## 3. PUT Update Address Details
Update existing address and location details of the logged-in user.

* **URL**: `/api/auth/profile/address`
* **Method**: `PUT`
* **Authentication Required**: Yes

### Request Body
* `pincode` is **optional**. If not provided (or empty/whitespace), it defaults internally to `0`.

```json
{
  "country": "India",
  "state": "Maharashtra",
  "city": "Mumbai",
  "addressLine1": "XYZ Marg",
  "addressLine2": "Apartment 12B"
}
```

### Success Response
* **Status Code**: `200 OK`

```json
{
  "statusCode": 200,
  "messageCode": "success.ADDRESS_UPDATED",
  "message": "Address details updated successfully"
}
```

### Error Responses
* **Status Code**: `404 Not Found` (Address record does not exist)

```json
{
  "statusCode": 404,
  "messageCode": "error.addressDetailsNotFound",
  "message": "Address details not found."
}
```

---

## 4. DELETE Remove Address Details
Hard-delete user address details from `tbl_user_address_details` while keeping location columns in `tbl_user_primary_details` intact.

* **URL**: `/api/auth/profile/address`
* **Method**: `DELETE`
* **Authentication Required**: Yes

### Success Response
* **Status Code**: `200 OK`

```json
{
  "statusCode": 200,
  "messageCode": "success.ADDRESS_DELETED",
  "message": "Address details deleted successfully"
}
```

### Error Responses
* **Status Code**: `404 Not Found` (Address record does not exist)

```json
{
  "statusCode": 404,
  "messageCode": "error.addressDetailsNotFound",
  "message": "Address details not found."
}
```
