# 19. Profile Gallery (Complete Master Specification)

This documentation provides the **complete, unified workflow** for the frontend team to implement all Profile Gallery and Profile Picture features.

---

## Complete Workflow Checklist

1. **[POST] Upload Physical Image(s):** `/api/file/upload-files`
2. **[POST] Sync Profile Gallery:** `/api/auth/user/gallery` (Session-based)
3. **[GET] Get Complete Gallery:** `/api/auth/user/gallery/{user_id}` (URL path-parameter)
4. **[GET] Get Active Profile Picture:** `/api/auth/user/getprofilepicture/{user_id}` (URL path-parameter)
5. **[PUT] Update Active Profile Picture:** `/api/auth/user/updateprofilepicture` (Session-based)
6. **[DELETE] Delete Gallery Image:** `/api/auth/user/gallery/{fileName}` (URL path-parameter)

---

## Step 1: Upload Physical Files (Multipart Upload)

Uploads raw image files to AWS S3 storage and generates secure, unique filename keys. 

* **URL:** `/api/file/upload-files`
* **Method:** `POST`
* **Headers:**
  ```http
  Content-Type: multipart/form-data
  ```
* **Request Body (form-data):**
  - Key: `image` (Type: `File`, Value: `[Raw Selected Image]`)

* **Sample Response (Success - 200 OK):**
  ```json
  [
      {
          "fileName": "0be142979cac4179b7e931c46fd747b0.jpg",
          "originalFileName": "my_profile_photo.jpg"
      }
  ]
  ```

---

## Step 2: Sync Profile Gallery (Save / Update Mapping)

Associates the uploaded S3 `fileName`s to the user's account.

> [!IMPORTANT]
> This API is a **sync (full overwrite)** transaction. On every save, you must send the **complete list** of active images that the user wants to retain. Any image previously saved but omitted from the new array will be automatically unlinked and removed.

> [!TIP]
> If multiple images are provided but none are marked with `"isProfilePicture": true`, the backend will **automatically fallback** and mark the **first image in the array** as the profile picture.

* **URL:** `/api/auth/user/gallery`
* **Method:** `POST`
* **Headers:**
  ```http
  Authorization: Bearer <user_token>
  x-app-type: user
  Content-Type: application/json
  Accept-Language: en-US
  ```
* **Request Body (JSON Array):**
  ```json
  [
    {
      "fileName": "0be142979cac4179b7e931c46fd747b0.jpg",
      "isProfilePicture": true
    },
    {
      "fileName": "6d06edb1f03142a5a0809c126375b7c9.jpg",
      "isProfilePicture": false
    }
  ]
  ```
* **Sample Response (Success - 200 OK):**
  ```json
  {
      "message": "Data saved successfully.",
      "statusCode": 200,
      "messageCode": "success.dataSaved"
  }
  ```

---

## Step 3: Get Complete Gallery (Retrieve Mapped Images)

Fetches the list of all active gallery images for any user ID. This is fully optimized for the Admin Panel and profile detail cards.

* **URL:** `/api/auth/user/gallery/{user_id}`
* **Method:** `GET`
* **Headers:**
  ```http
  Authorization: Bearer <user_token>
  x-app-type: user / admin
  Accept-Language: en-US
  ```
* **Sample Response (Success - 200 OK):**
  *(Returns raw unwrapped JSON array of active images directly)*
  ```json
  [
      {
          "fileName": "0be142979cac4179b7e931c46fd747b0.jpg",
          "isProfilePicture": true
      },
      {
          "fileName": "6d06edb1f03142a5a0809c126375b7c9.jpg",
          "isProfilePicture": false
      }
  ]
  ```

---

## Step 4: Get Active Profile Picture

Fetches only the singular S3 filename of the active profile picture for the specified user ID.

* **URL:** `/api/auth/user/getprofilepicture/{user_id}`
* **Method:** `GET`
* **Headers:**
  ```http
  Authorization: Bearer <user_token>
  x-app-type: user / admin
  Accept-Language: en-US
  ```
* **Sample Response (Success - 200 OK):**
  *(Returns raw unwrapped JSON dictionary directly)*
  ```json
  {
      "profilePicture": "0be142979cac4179b7e931c46fd747b0.jpg"
  }
  ```

---

## Step 5: Update Active Profile Picture

Switches the user's active profile picture to a different image using its unique S3 `fileName`.

* **URL:** `/api/auth/user/updateprofilepicture`
* **Method:** `PUT`
* **Headers:**
  ```http
  Authorization: Bearer <user_token>
  x-app-type: user
  Content-Type: application/json
  Accept-Language: en-US
  ```
* **Request Body (JSON):**
  ```json
  {
      "fileId": "6d06edb1f03142a5a0809c126375b7c9.jpg"
  }
  ```
* **Sample Response (Success - 200 OK):**
  ```json
  {
      "message": "Data saved successfully.",
      "statusCode": 200,
      "messageCode": "success.dataSaved"
  }
  ```

---

## Step 6: Delete Gallery Image by Filename

Soft-deletes a specific image from the user's gallery using its S3 `fileName`.

> [!IMPORTANT]
> **Automatic Fallback Promotion:** If the deleted image was currently selected as the active profile picture (`is_profile_picture = TRUE`), the backend automatically finds the user's next oldest active (non-deleted) image in their gallery and promotes it to `is_profile_picture = TRUE` so their profile photo never remains blank.

* **URL:** `/api/auth/user/gallery/{fileName}`
* **Method:** `DELETE`
* **Headers:**
  ```http
  Authorization: Bearer <user_token>
  x-app-type: user
  Accept-Language: en-US
  ```
* **Sample Response (Success - 200 OK):**
  ```json
  {
      "message": "Data deleted successfully.",
      "statusCode": 200,
      "messageCode": "success.dataDeleted"
  }
  ```

## Step 7: Render Images in the Frontend

Render the retrieved gallery images using the download service endpoints. **No authentication headers are required** for download requests, allowing them to be loaded directly inside image tags!

### 1. Fast Preview / Thumbnails (Highly Recommended!)
Our backend automatically generates optimized, lightweight thumbnails for every uploaded image. **Always use this for lists, dashboard cards, and grid previews** for high-speed loading.

To load the thumbnail, prefix the S3 `fileName` with **`thumbnail_`**:
* **URL:** `http://bandhan-setu-prod.eba-am6hwsad.ap-south-1.elasticbeanstalk.com/api/file/download/thumbnail_<fileName>`
* **HTML Example:**
  ```html
  <img src="http://bandhan-setu-prod.eba-am6hwsad.ap-south-1.elasticbeanstalk.com/api/file/download/thumbnail_0be142979cac4179b7e931c46fd747b0.jpg" />
  ```

### 2. High-Quality Original Images
Load the original high-resolution photo on the profile detail/expanded screen.
* **URL:** `http://bandhan-setu-prod.eba-am6hwsad.ap-south-1.elasticbeanstalk.com/api/file/download/<fileName>`
* **HTML Example:**
  ```html
  <img src="http://bandhan-setu-prod.eba-am6hwsad.ap-south-1.elasticbeanstalk.com/api/file/download/0be142979cac4179b7e931c46fd747b0.jpg" />
  ```
