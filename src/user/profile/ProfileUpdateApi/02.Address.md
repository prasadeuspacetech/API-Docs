# Address Details API Documentation

This document provides a comprehensive specification of the Address Details API endpoints implemented in the backend.

---

## 🔑 Global Request Headers

All requests to the Address Details endpoints must pass through the `BaseHandler` middleware, which enforces security, language translation, and application-type validation. The following headers are **required** for every endpoint:

| Header Name | Required | Type | Value / Format | Description |
| :--- | :--- | :--- | :--- | :--- |
| `Accept-Language` | **Yes** | String | e.g. `en-US`, `hi`, `mr` | Language code for localization of success/error messages. |
| `Authorization` | **Yes** | String | `Bearer <JWT_Token>` | User authentication token. |
| `x-app-type` | **Yes** | String | `user` | Required application context. Only requests from `user` apps are authorized. |

---

## 🛠️ Endpoints

### 1. Get Address Details
Retrieves the address details of a specific user.

*   **HTTP Method**: `GET`
*   **Path**: `/api/auth/profile/address/{userId}`
*   **Path Parameters**:
    *   `userId` (String, Required): The unique UUID of the user whose address details you want to retrieve.
*   **Request Body**: None

#### Success Response (`200 OK`)
*   **Content-Type**: `application/json`
*   **Response Body**: Returns the **raw JSON object** (not wrapped in a generic message envelope):
    ```json
    {
      "country": "India",
      "state": "Maharashtra",
      "city": "Mumbai",
      "addressLine1": "Flat 101, Sunny Apartments",
      "addressLine2": "Main Street",
      "pincode": 400001
    }
    ```
    *(Note: If the user has no registered address details, fields return as empty strings `""` and pincode returns as `0`.)*

#### Database Source Mapping:
*   `country`, `state`, and `city` are pulled from the user's primary details (`tbl_user_primary_details.country_name`, `tbl_user_primary_details.state_name`, `tbl_user_primary_details.city_name`).
*   `addressLine1`, `addressLine2`, and `pincode` are pulled from the user's address details (`tbl_user_address_details.address_line1`, `tbl_user_address_details.address_line2`, `tbl_user_address_details.pincode`).

---

### 2. Create Address Details
Creates address details for the currently logged-in user session.

*   **HTTP Method**: `POST`
*   **Path**: `/api/auth/profile/address`
*   **Request Body**:
    *   **Content-Type**: `application/json`
    *   **Parameters**:
        
        | Parameter Name | Required | Type | Description |
        | :--- | :--- | :--- | :--- |
        | `addressLine1` | **Yes** | String | First line of the address. Cannot be empty or whitespace. |
        | `addressLine2` | No | String | Second line of the address. |

    *   **Example Payload**:
        ```json
        {
          "addressLine1": "Flat 101, Sunny Apartments",
          "addressLine2": "Main Street"
        }
        ```

#### Success Response (`200 OK`)
*   **Content-Type**: `application/json`
*   **Response Body**:
    ```json
    {
      "message": "Address details created successfully",
      "statusCode": 200,
      "messageCode": "success.ADDRESS_CREATED"
    }
    ```

#### Key Actions & Business Logic:
1.  **Duplicate Check**: Verifies if address details already exist for this user in the database (`tbl_user_address_details`). If they exist, returns `409 Conflict`.
2.  **Creation**: Inserts the new record into the DB with a generated UUID, `created_by` and `created_at` timestamp.
3.  **Completeness State**: Automatically flags the user's profile completeness status for address details as `true` in `tbl_user_profile_completeness`.

---

### 3. Update Address Details
Updates address lines for the currently logged-in user session.

*   **HTTP Method**: `PUT`
*   **Path**: `/api/auth/profile/address`
*   **Request Body**:
    *   **Content-Type**: `application/json`
    *   **Parameters**:
        
        | Parameter Name | Required | Type | Description |
        | :--- | :--- | :--- | :--- |
        | `addressLine1` | **Yes** | String | Updated first line of the address. Cannot be empty or whitespace. |
        | `addressLine2` | No | String | Updated second line of the address. |

    *   **Example Payload**:
        ```json
        {
          "addressLine1": "Flat 102, Sunny Apartments",
          "addressLine2": "Main Street"
        }
        ```

#### Success Response (`200 OK`)
*   **Content-Type**: `application/json`
*   **Response Body**:
    ```json
    {
      "message": "Address details updated successfully",
      "statusCode": 200,
      "messageCode": "success.ADDRESS_UPDATED"
    }
    ```

#### Key Actions & Business Logic:
1.  **Existence Check**: Checks if the address record exists for the user in `tbl_user_address_details`. If no active record exists, returns `404 Not Found`.
2.  **Modification**: Updates the address fields (`address_line1`, `address_line2`) and sets the `updated_by` and `updated_at` properties.
3.  **Completeness State**: Ensures the profile completeness flag for address details remains `true`.

---

### 4. Delete Address Details
Deletes the address details for the currently logged-in user.

*   **HTTP Method**: `DELETE`
*   **Path**: `/api/auth/profile/address`
*   **Request Body**: None

#### Success Response (`200 OK`)
*   **Content-Type**: `application/json`
*   **Response Body**:
    ```json
    {
      "message": "Address details deleted successfully",
      "statusCode": 200,
      "messageCode": "success.ADDRESS_DELETED"
    }
    ```

#### Key Actions & Business Logic:
1.  **Existence Check**: Checks if the address record exists for the user in `tbl_user_address_details`. If not found, returns `404 Not Found`.
2.  **Hard Deletion**: Performs a SQL `DELETE` query against `tbl_user_address_details` for the authenticated user ID.

## 🛑 Common Error Responses

The following standardized JSON structures are returned by the application when requests fail:

### Accept-Language Header Missing (`400 Bad Request`)
```json
{
  "message": "Accept-Language header is required.",
  "statusCode": 400,
  "messageCode": "error.acceptLanguageRequired"
}
```

### Invalid JSON Body (`400 Bad Request`)
```json
{
  "message": "Invalid JSON body.",
  "statusCode": 400,
  "messageCode": "error.invalidJson"
}
```

### Missing Required Field (`400 Bad Request`)
```json
{
  "message": "addressLine1 is required.",
  "statusCode": 400,
  "messageCode": "error.fieldRequired"
}
```

### Authentication Token Missing (`401 Unauthorized`)
```json
{
  "message": "Authorization token is required.",
  "statusCode": 401,
  "messageCode": "error.tokenRequired"
}
```

### Invalid / Expired Token (`401 Unauthorized`)
```json
{
  "message": "Invalid or expired token.",
  "statusCode": 401,
  "messageCode": "error.invalidToken"
}
```

### Incorrect Application Type Context (`403 Forbidden`)
```json
{
  "message": "This endpoint requires x-app-type: user header.",
  "statusCode": 403,
  "messageCode": "error.userAppTypeRequired"
}
```

### Address Record Not Found (`404 Not Found`)
*Returned on PUT/DELETE if no active address record exists for the user.*
```json
{
  "message": "Address details not found.",
  "statusCode": 404,
  "messageCode": "error.addressDetailsNotFound"
}
```

### Address Record Already Exists (`409 Conflict`)
*Returned on POST if address record already exists for the user.*
```json
{
  "message": "Address details already exist.",
  "statusCode": 409,
  "messageCode": "error.addressDetailsExists"
}
```
