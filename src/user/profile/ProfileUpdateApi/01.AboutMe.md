# About Me API Documentation

This document describes the API endpoints for fetching and updating the "About Me" section of a user profile.

---

## 1. Get About Me Details

Fetches the "About Me" text content for a specified user profile.

* **Endpoint**: `GET /api/auth/profile/about-me/{userId}`
* **Authentication**: Required (JWT Bearer Token)
* **Request Headers**:
  * `Authorization`: `Bearer <token>`
  * `x-app-type`: `user`
  * `accept-language`: `en-US` (or `mr`, `hi` for localized messages)

### Request Parameters

* **Path Parameters**:
  * `userId` (string, required): The UUID of the user whose "About Me" details you want to retrieve.

### Success Response

* **Status Code**: `200 OK`
* **Content-Type**: `application/json`
* **Response Body**:
  ```json
  {
    "aboutMe": "I love coding and building APIs."
  }
  ```
  *(Note: If the user has not yet populated their "About Me" section or if the record does not exist, an empty string `""` is returned instead).*

### Error Responses

* **400 Bad Request** (Missing/invalid headers):
  ```json
  {
    "statusCode": 400,
    "messageCode": "error.acceptLanguageRequired",
    "message": "Accept-Language header is required."
  }
  ```
* **401 Unauthorized** (Missing or invalid token):
  ```json
  {
    "statusCode": 401,
    "messageCode": "error.invalidToken",
    "message": "Invalid or expired token."
  }
  ```

---

## 2. Update About Me Details

Updates the "About Me" text content for the currently authenticated user's profile.

* **Endpoint**: `PUT /api/auth/profile/about-me`
* **Authentication**: Required (JWT Bearer Token)
* **Request Headers**:
  * `Authorization`: `Bearer <token>`
  * `x-app-type`: `user`
  * `accept-language`: `en-US` (or `mr`, `hi` for localized messages)

### Request Body

* **Content-Type**: `application/json`
* **Payload**:
  ```json
  {
    "aboutMe": "I love coding and building APIs"
  }
  ```

### Validation Rules

* `aboutMe` is required, must be a string, and cannot be empty or contain only whitespace.

### Success Response

* **Status Code**: `200 OK`
* **Content-Type**: `application/json`
* **Response Body**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "ABOUT_ME_UPDATED",
    "message": "About me updated successfully"
  }
  ```

### Localized Success Responses

Depending on the `accept-language` header sent:

* **Marathi (`accept-language: mr`)**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "ABOUT_ME_UPDATED",
    "message": "माझ्याबद्दल माहिती यशस्वीरित्या अद्यतनित केली."
  }
  ```
* **Hindi (`accept-language: hi`)**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "ABOUT_ME_UPDATED",
    "message": "मेरे बारे में जानकारी सफलतापूर्वक अपडेट की गई।"
  }
  ```

### Error Responses

* **400 Bad Request** (Validation failure / missing body field):
  ```json
  {
    "statusCode": 400,
    "messageCode": "error.fieldRequired",
    "message": "aboutMe is required."
  }
  ```
* **404 Not Found** (User does not have an active setup profile / record in database):
  ```json
  {
    "statusCode": 404,
    "messageCode": "error.userNotFound",
    "message": "User not found."
  }
  ```
