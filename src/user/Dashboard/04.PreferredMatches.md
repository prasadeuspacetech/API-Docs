# API Documentation: Preferred Matches

This documentation details the endpoints for managing and retrieving a user's **Preferred Matches**.

---

## Headers Required for All Requests

| Header Name | Required | Expected Value | Description |
| :--- | :--- | :--- | :--- |
| `Authorization` | Yes | `Bearer <JWT_TOKEN>` | JSON Web Token (JWT) retrieved on successful login. |
| `x-app-type` | Yes | `user` | Required application identifier context. |
| `Accept-Language` | Yes | `en-US` (or locale code) | Standard language preference code. |

---

## 1. Mark Preferred Match

Updates the preference state for a specific friend connection. Marking a match as preferred moves them from the standard Accepted list to the Preferred Matches list.

* **URL Path:** `/api/auth/user/friend-request/preferred`
* **HTTP Method:** `PUT`
* **Content-Type:** `application/json`

### Request Body
```json
{
  "target_user_id": "070cd6f0e992430fbf66283543ef60ff",
  "is_preferred": 1
}
```

#### Fields Description
* `target_user_id` (string, required): The hex ID of the connected user.
* `is_preferred` (integer, required): Preference flag. Use `1` to mark as preferred, `0` to unmark.

### Success Response (`200 OK`)
```json
{
  "status": "success",
  "message": "Friend request status updated successfully.",
  "data": {}
}
```

### Error Responses

#### `400 Bad Request` (Invalid JSON)
```json
{
  "status": "error",
  "message": "Invalid JSON body.",
  "data": {}
}
```

#### `400 Bad Request` (Invalid Inputs / Missing Fields)
```json
{
  "status": "error",
  "message": "target_user_id is required.",
  "data": {}
}
```
*(or if `is_preferred` is not `0` or `1`)*:
```json
{
  "status": "error",
  "message": "is_preferred must be an integer (0 or 1).",
  "data": {}
}
```

#### `401 Unauthorized` (Token Missing/Invalid)
```json
{
  "status": "error",
  "message": "Invalid or expired token.",
  "data": {}
}
```

#### `403 Forbidden` (App Context Incorrect)
```json
{
  "status": "error",
  "message": "This endpoint requires x-app-type: user header.",
  "data": {}
}
```

#### `404 Not Found` (No accepted relationship exists)
```json
{
  "status": "error",
  "message": "No active friend request found from this user.",
  "data": {}
}
```

## 2. Get Preferred Matches List

Retrieves a paginated list of all connected users who are currently marked as preferred matches.

* **URL Path:** `/api/auth/user/friend-request/preferred`
* **HTTP Method:** `GET`

### Query Parameters
* `page` (integer, optional, default: `1`): The page number.
* `limit` (integer, optional, default: `5`, max: `20`): Number of items to fetch per page.

### Success Response (`200 OK`)
```json
{
  "status": "success",
  "message": "Data fetched successfully.",
  "data": [
    {
      "id": "e64ff0f1953543d7a4b68e12dbe01d53",
      "connected_user_id": "070cd6f0e992430fbf66283543ef60ff",
      "status": "Accepted",
      "createdAt": 1716912345,
      "platformId": "BSU12345",
      "age": 28,
      "educationLevelName": "B.Tech",
      "annualIncome": "10-15 LPA",
      "religionName": "Hindu",
      "castName": "Maratha",
      "cityName": "Pune",
      "stateName": "Maharashtra"
    }
  ]
}
```

#### Field Details in `data` array:
* `id`: The friendship/request record ID (`tbl_user_friend_request.id`). Keep this row ID to track the relation record.
* `connected_user_id`: The user ID of the matched profile.
* `createdAt`: Epoch timestamp (integer) of when the connection request was created.
* Profile attributes: `platformId`, `age`, `educationLevelName`, `annualIncome`, `religionName`, `castName`, `cityName`, `stateName`.

### Error Responses

#### `401 Unauthorized` (Token Missing/Invalid)
```json
{
  "status": "error",
  "message": "Invalid or expired token.",
  "data": {}
}
```
