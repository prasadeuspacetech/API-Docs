# Lifestyle Details API Documentation

This document describes the API endpoints for fetching and updating the "Lifestyle Details" section of a user profile.

---

## 1. Get Lifestyle Details

Retrieves the lifestyle details of a specified user profile.

* **Endpoint**: `GET /api/auth/profile/lifestyle/{userId}`
* **Authentication**: Required (JWT Bearer Token)
* **Request Headers**:
  * `Authorization`: `Bearer <token>`
  * `x-app-type`: `user`
  * `accept-language`: `en-US` (or `mr`, `hi` for localized messages)

### Request Parameters

* **Path Parameters**:
  * `userId` (string, required): The UUID of the user whose lifestyle details you want to retrieve.

### Success Response

* **Status Code**: `200 OK`
* **Content-Type**: `application/json`
* **Response Body**:
  ```json
  {
    "diet": "Vegetarian",
    "smoke": "No",
    "drink": "Occasionally",
    "bodyType": "Athletic",
    "userWeight": 72
  }
  ```
  *(Note: If the active record does not exist for the user, empty strings `""` are returned for all fields, and `userWeight` defaults to `0`).*

### Error Responses

* **401 Unauthorized** (Missing or invalid token):
  ```json
  {
    "statusCode": 401,
    "messageCode": "error.invalidToken",
    "message": "Invalid or expired token."
  }
  ```

---

## 2. Update Lifestyle Details

Updates the lifestyle details of the currently authenticated user.

* **Endpoint**: `PUT /api/auth/profile/lifestyle`
* **Authentication**: Required (JWT Bearer Token)
* **Request Headers**:
  * `Authorization`: `Bearer <token>`
  * `x-app-type`: `user`
  * `accept-language`: `en-US` (or `mr`, `hi` for localized messages)

### Request Body

* **Content-Type**: `application/json`
* **Payload**:
  ```json
  {
    "diet": "Vegetarian",
    "smoke": "No",
    "drink": "Occasionally",
    "bodyType": "Athletic",
    "userWeight": 72
  }
  ```
  *(Note: `skinTone` and `userHeight` are ignored and will NOT be modified in the database).*

### Validation Rules

* `diet`, `smoke`, `drink`, `bodyType`, and `userWeight` are required fields and must be non-empty.
* `smoke` must be one of: `"Yes"`, `"No"`, or `"Occasionally"`.
* `drink` must be one of: `"Yes"`, `"No"`, or `"Occasionally"`.
* `userWeight` must be a valid positive integer (greater than 0).

### Success Response

* **Status Code**: `200 OK`
* **Content-Type**: `application/json`
* **Response Body**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "success.LIFESTYLE_UPDATED",
    "message": "Lifestyle details updated successfully"
  }
  ```

### Localized Success Responses

Depending on the `accept-language` header sent:

* **Marathi (`accept-language: mr`)**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "success.LIFESTYLE_UPDATED",
    "message": "जीवनशैलीचा तपशील यशस्वीरित्या अद्यतनित केला."
  }
  ```
* **Hindi (`accept-language: hi`)**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "success.LIFESTYLE_UPDATED",
    "message": "जीवनशैली विवरण सफलतापूर्वक अपडेट किया गया।"
  }
  ```

### Error Responses

* **400 Bad Request** (Malformed Request Payload / Invalid JSON):
  ```json
  {
    "statusCode": 400,
    "messageCode": "error.invalidJson",
    "message": "Invalid JSON data."
  }
  ```
* **400 Bad Request** (Missing/Empty Required Fields):
  ```json
  {
    "statusCode": 400,
    "messageCode": "error.fieldRequired",
    "message": "diet is required."
  }
  ```
* **400 Bad Request** (Invalid Choice/Value for Smoke/Drink/Weight):
  ```json
  {
    "statusCode": 400,
    "messageCode": "error.smokeInvalid",
    "message": "smoke must be Yes, No or Occasionally."
  }
  ```
  *(Or `error.weightInvalid` for invalid userWeight).*
* **404 Not Found** (Active lifestyle details record not found / user not registered):
  ```json
  {
    "statusCode": 404,
    "messageCode": "error.userNotFound",
    "message": "User not found."
  }
  ```
* **401 Unauthorized** (Missing or invalid token):
  ```json
  {
    "statusCode": 401,
    "messageCode": "error.invalidToken",
    "message": "Invalid or expired token."
  }
  ```
