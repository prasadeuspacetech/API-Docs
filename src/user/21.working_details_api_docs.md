# Working Details API Documentation

This document describes the API endpoints for fetching and updating the "Working Details" section of a user profile.

---

## 1. Get Working Details

Retrieves the working details of a specified user profile.

* **Endpoint**: `GET /api/auth/profile/working-details/{userId}`
* **Authentication**: Required (JWT Bearer Token)
* **Request Headers**:
  * `Authorization`: `Bearer <token>`
  * `x-app-type`: `user`
  * `accept-language`: `en-US` (or `mr`, `hi` for localized messages)

### Request Parameters

* **Path Parameters**:
  * `userId` (string, required): The UUID of the user whose working details you want to retrieve.

### Success Response

* **Status Code**: `200 OK`
* **Content-Type**: `application/json`
* **Response Body**:
  ```json
  {
    "employerName": "Infosys",
    "annualIncomeName": "10-15 Lakhs",
    "workingWithName": "Private Company",
    "currencyType": "INR",
    "subcategoryName": "Software Engineer",
    "categoryName": "IT"
  }
  ```
  *(Note: If the active record does not exist for the user, empty strings `""` are returned for all fields, except `currencyType` which defaults to `"INR"`).*

### Error Responses

* **401 Unauthorized** (Missing or invalid token):
  ```json
  {
    "statusCode": 401,
    "messageCode": "error.invalidToken",
    "message": "Invalid or expired token."
  }
  ```

---

## 2. Update Working Details

Updates the working details of the currently authenticated user.

* **Endpoint**: `PUT /api/auth/profile/working-details`
* **Authentication**: Required (JWT Bearer Token)
* **Request Headers**:
  * `Authorization`: `Bearer <token>`
  * `x-app-type`: `user`
  * `accept-language`: `en-US` (or `mr`, `hi` for localized messages)

### Request Body

* **Content-Type**: `application/json`
* **Payload**:
  ```json
  {
    "employerName": "Infosys",
    "annualIncomeName": "10-15 Lakhs",
    "workingWithName": "Private Company",
    "subcategoryName": "Software Engineer",
    "categoryName": "IT"
  }
  ```
  *(Note: `currencyType` is locked to `"INR"` in the database and must not be updated. Any incoming `currencyType` field in the payload will be ignored).*

### Validation Rules

* `employerName`, `annualIncomeName`, `workingWithName`, `subcategoryName`, and `categoryName` are required fields and must be non-empty strings.

### Success Response

* **Status Code**: `200 OK`
* **Content-Type**: `application/json`
* **Response Body**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "success.WORKING_DETAILS_UPDATED",
    "message": "Working details updated successfully"
  }
  ```

### Localized Success Responses

Depending on the `accept-language` header sent:

* **Marathi (`accept-language: mr`)**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "success.WORKING_DETAILS_UPDATED",
    "message": "कामकाजाचा तपशील यशस्वीरित्या अद्यतनित केला."
  }
  ```
* **Hindi (`accept-language: hi`)**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "success.WORKING_DETAILS_UPDATED",
    "message": "कामकाज विवरण सफलतापूर्वक अपडेट किया गया।"
  }
  ```

### Error Responses

* **400 Bad Request** (Malformed Request Payload / Invalid JSON):
  ```json
  {
    "statusCode": 400,
    "messageCode": "error.invalidJson",
    "message": "Invalid JSON data."
  }
  ```
* **400 Bad Request** (Missing/Empty Required Fields):
  ```json
  {
    "statusCode": 400,
    "messageCode": "error.fieldRequired",
    "message": "employerName is required."
  }
  ```
* **404 Not Found** (Active working details record not found / user not registered):
  ```json
  {
    "statusCode": 404,
    "messageCode": "error.userNotFound",
    "message": "User not found."
  }
  ```
* **401 Unauthorized** (Missing or invalid token):
  ```json
  {
    "statusCode": 401,
    "messageCode": "error.invalidToken",
    "message": "Invalid or expired token."
  }
  ```
