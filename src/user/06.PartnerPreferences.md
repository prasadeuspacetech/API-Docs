# 06. Partner Preferences (Screen 4)

This API is used to manage the user's preferences for their ideal partner. It supports separate methods for creation and updates.

---

## 1. Create Partner Preferences

This API is used to save the user's preferences for the first time. It will fail if preferences already exist.

**URL:**
`/api/auth/user/user-partner-preferences`

**Method:**
`POST`

**Headers:**
```
Authorization: Bearer {{token}}
x-app-type: user
Content-Type: application/json
Accept-Language: en-US
```

**Sample Input (camelCase)**

```json
{
    "ageFrom": 20,
    "ageTo": 30,
    "heightFromFeet": 5,
    "heightFromInches": 0,
    "heightToFeet": 6,
    "heightToInches": 2,
    "maritalStatus": ["Never Married", "Divorced"],
    "religionName": ["Hindu", "Sikh"],
    "castName": ["Brahmin", "Kshatriya"],
    "subcastName": ["Deshastha", "Kokanastha"],
    "mothertongueName": ["Hindi", "Marathi", "English"],
    "countryName": ["India", "United Arab Emirates"],
    "stateName": ["Maharashtra", "Gujarat"],
    "cityName": ["Pune", "Mumbai", "Surat"],
    "educationLevelName": ["Bachelor's", "Master's"],
    "educationFieldName": ["Computer Science", "Finance"],
    "workingWithName": ["Private Sector", "Self Employed"],
    "workingWithCategoryName": ["Software Professional", "Banking"],
    "subcategoryName": ["Backend Developer", "Relationship Manager"]
}
```

**Sample Output (Success)**

```json
{
    "message": "Partner preferences saved successfully.",
    "statusCode": 200,
    "messageCode": "success.partnerPreferencesSaved",
    "profileCompleteness": {
        "id": "00ddbfc79419419ab3ad63b2df24a774",

```

**Error Message Scenarios**

1. Preferences Already Exist
```json
{
  "message": "Partner preferences already exist.",
  "statusCode": 409,
  "messageCode": "error.partnerPreferencesExists"
}
```

---

## 2. Update Partner Preferences

This API is used to update existing preferences. It supports partial updates (you can send only the fields you want to change). It will fail if preferences do not exist.

**URL:**
`/api/auth/user/user-partner-preferences`

**Method:**
`PATCH`

**Headers:**
```
Authorization: Bearer {{token}}
x-app-type: user
Content-Type: application/json
Accept-Language: en-US
```

**Sample Input (camelCase)**
*(Send only fields to update)*
```json
{
    "ageFrom": 22,
    "ageTo": 32
}
```

**Sample Output (Success)**
*(Same as POST)*

**Error Message Scenarios**

1. Preferences Not Found
```json
{
  "message": "Partner preferences not found.",
  "statusCode": 404,
  "messageCode": "error.partnerPreferencesNotFound"
}

```

**URL:**
`/api/auth/user/user-partner-preferences`

**Method:**
