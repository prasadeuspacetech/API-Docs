# Education Details API Documentation

This document describes the API endpoints for fetching and updating the "Education Details" section of a user profile.

---

## 1. Get Education Details

Retrieves the education details of a specified user profile.

* **Endpoint**: `GET /api/auth/profile/education/{userId}`
* **Authentication**: Required (JWT Bearer Token)
* **Request Headers**:
  * `Authorization`: `Bearer <token>`
  * `x-app-type`: `user`
  * `accept-language`: `en-US` (or `mr`, `hi` for localized messages)

### Request Parameters

* **Path Parameters**:
  * `userId` (string, required): The UUID of the user whose education details you want to retrieve.

### Success Response

* **Status Code**: `200 OK`
* **Content-Type**: `application/json`
* **Response Body**:
  ```json
  {
    "collegeName": "MIT College",
    "educationLevelName": "Bachelor's",
    "educationFieldName": "Computer Science"
  }
  ```
  *(Note: If the active record does not exist for the user, empty strings `""` are returned for all fields).*

### Error Responses

* **401 Unauthorized** (Missing or invalid token):
  ```json
  {
    "statusCode": 401,
    "messageCode": "error.invalidToken",
    "message": "Invalid or expired token."
  }
  ```

---

## 2. Update Education Details

Updates the education details of the currently authenticated user.

* **Endpoint**: `PUT /api/auth/profile/education`
* **Authentication**: Required (JWT Bearer Token)
* **Request Headers**:
  * `Authorization`: `Bearer <token>`
  * `x-app-type`: `user`
  * `accept-language`: `en-US` (or `mr`, `hi` for localized messages)

### Request Body

* **Content-Type**: `application/json`
* **Payload**:
  ```json
  {
    "collegeName": "MIT College",
    "educationLevelName": "Bachelor's",
    "educationFieldName": "Computer Science"
  }
  ```

### Validation Rules

* `collegeName`, `educationLevelName`, and `educationFieldName` are required fields and must be non-empty strings.

### Success Response

* **Status Code**: `200 OK`
* **Content-Type**: `application/json`
* **Response Body**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "success.EDUCATION_UPDATED",
    "message": "Education details updated successfully"
  }
  ```

### Localized Success Responses

Depending on the `accept-language` header sent:

* **Marathi (`accept-language: mr`)**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "success.EDUCATION_UPDATED",
    "message": "शिक्षण तपशील यशस्वीरित्या अद्यतनित केले."
  }
  ```
* **Hindi (`accept-language: hi`)**:
  ```json
  {
    "statusCode": 200,
    "messageCode": "success.EDUCATION_UPDATED",
    "message": "शिक्षा विवरण सफलतापूर्वक अपडेट किया गया।"
  }
  ```

### Error Responses

* **400 Bad Request** (Malformed Request Payload / Invalid JSON):
  ```json
  {
    "statusCode": 400,
    "messageCode": "error.invalidJson",
    "message": "Invalid JSON data."
  }
  ```
* **400 Bad Request** (Missing/Empty Required Fields):
  ```json
  {
    "statusCode": 400,
    "messageCode": "error.fieldRequired",
    "message": "collegeName is required."
  }
  ```
* **404 Not Found** (Active education record not found / user not registered):
  ```json
  {
    "statusCode": 404,
    "messageCode": "error.userNotFound",
    "message": "User not found."
  }
  ```
* **401 Unauthorized** (Missing or invalid token):
  ```json
  {
    "statusCode": 401,
    "messageCode": "error.invalidToken",
    "message": "Invalid or expired token."
  }
  ```
